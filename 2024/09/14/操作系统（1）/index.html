第一章 绪论 操作系统的基本概念

定义： 操作系统（Operating System, OS）是管理计算机硬件和软件资源的系统软件，它作为用户与计算机硬件之间的桥梁。操作系统的主要功能包括：

  1. 进程管理：管理程序的执行，包括进程的创建、调度和终止。操作系统确保每个进程在合适的时间得到合适的资源，同时避免冲突。

  2. 内存管理：负责内存的分配与回收，确保多个程序能够在内存中同时执行，并且不会相互干扰。

  3. 文件系统管理：提供对文件的创建、读取、写入、删除等操作，并管理文件的存储位置和权限控制。

  4. 设备管理：管理计算机的输入输出设备（如硬盘、显示器、键盘等），通过驱动程序与硬件交互，并提供标准接口供应用程序使用。

  5. 用户界面：为用户提供与计算机交互的界面，常见的有命令行界面（CLI）和图形用户界面（GUI）。

常见的操作系统有Windows、macOS、Linux、Android等。它们的核心部分称为“内核”（Kernel），负责控制系统的各个方面。 监督程序（Supervisor Program）是操作系统的核心组成部分，负责控制和管理计算机系统的基本操作。它通常运行在内核模式下，直接与硬件交互，并管理用户程序的执行和系统资源的分配。

监督程序的主要功能：

  1. 进程管理：监督程序调度和管理所有正在运行的程序，确保每个程序都能按照系统的规则执行。它负责分配CPU时间给不同的进程，处理进程间的切换以及进程的创建和终止。

  2. 内存管理：监督程序负责分配和回收内存资源，防止不同程序之间的内存相互干扰。同时，它还可能负责虚拟内存管理，将物理内存与硬盘空间结合使用。

  3. I/O管理：监督程序管理所有输入/输出操作，包括与外部设备的交互。它通过设备驱动程序控制硬件资源，确保用户程序正确使用这些资源。

  4. 安全与保护：监督程序可以防止非法访问系统资源，提供用户权限管理和进程隔离功能，确保系统的稳定性和安全性。

  5. 异常处理：当系统出现错误或硬件故障时，监督程序负责捕捉并处理这些异常，防止系统崩溃，并进行必要的恢复或报错。

监督程序在操作系统中的作用：

监督程序可以看作是整个操作系统的“管理者”，它确保系统的稳定、高效运行，并为用户程序提供安全的执行环境。监督程序与用户态的程序不同，它拥有对系统资源的完全控制权限，负责在需要时对系统资源进行调度和分配。

在现代操作系统中，监督程序是内核的一部分，通过它的管理，操作系统能够以多任务、并发、安全的方式运行。

第二节课

批处理操作系统和分时操作系统的对比

下表比较了批处理操作系统和分时操作系统：

对比项 批处理操作系统 分时操作系统
任务处理方式 将多个任务按批次提交，系统依次处理，任务执行时无需用户干预。 通过时间片轮转调度，多个任务交替执行，支持多任务并发。
用户交互 用户无法与正在运行的任务交互，提交后等待结果。 支持用户与系统交互，用户可以实时提交任务并获取快速响应。
资源管理 任务按作业队列处理，依赖优先级和资源需求进行调度。 通过时间片分配，保障每个任务在一定时间内使用处理器资源。
响应时间 响应时间较长，任务完成后才能返回结果。 响应时间较短，任务在时间片内可获得处理。
系统开销 系统开销较低，因任务切换较少。 系统开销较高，因频繁的任务切换和资源分配。
适用场景 适用于批量、长时间运行且无需实时响应的任务（如数据处理）。 适用于多用户、多任务并发且需要实时响应的任务（如在线系统）。
资源利用率 资源利用率较高，系统资源能高效利用。 资源利用率相对较低，更多资源用于任务调度和管理。

Windows 操作系统尽管最初是为单用户设计的，但它实现了多任务和多用户支持的特性，使其具备了分时操作系统的特征。以下是几个体现 Windows 上分时操作系统机制的例子：

1. 多任务处理

  • Windows 支持并发多任务，允许多个程序在同一时间运行。通过时间片轮转调度算法，操作系统会在多个运行的任务之间快速切换，使用户感觉多个任务在同时进行。例如：
    • 用户可以同时运行 Word 文档、浏览器和音乐播放器，而系统会在这些应用程序之间分配 CPU 资源。

2. 多用户登录

  • 虽然 Windows 系统常常是单用户的环境，但通过 远程桌面服务（Remote Desktop Services），支持多个用户同时登录并使用系统。每个用户通过远程桌面协议（RDP）登录后，都有一个独立的会话，系统在多个用户会话之间切换并分配资源。

3. 时间片调度

  • Windows 使用 抢占式多任务调度，即每个进程分配到一段时间片（CPU 时间），当时间片用完后，操作系统会将 CPU 分配给其他进程。这样即便是多个用户程序同时运行，系统仍能公平地分配处理器时间，保障各任务的及时响应。

4. 虚拟内存和内存管理

  • Windows 使用虚拟内存技术来管理多任务时的内存需求。即使物理内存不足，系统会将暂时不需要的内存页换出到磁盘（页面文件），从而保证多个任务可以同时运行，而不占用过多的物理内存。

5. 线程调度

  • Windows 的 内核调度器 负责将 CPU 时间片分配给线程，线程是进程中的最小执行单元。通过线程调度，Windows 能够高效地处理多个应用程序的请求，保障系统的并发性和实时响应。

通过这些机制，Windows 具备了分时操作系统的基本特征，即多用户、多任务并发处理和时间片调度，尽管它是一个图形化、面向个人计算机的操作系统。
